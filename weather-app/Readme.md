# Weather Console App - Получение погоды через API

Консольное приложение на Node.js для получения актуальной информации о погоде с использованием API Яндекс.Погода.

## 📋 Описание проекта

Данное приложение разработано в рамках домашнего задания к занятию **«1.5. Http module. Переменные окружения»** курса по изучению Node.js.

### Задачи проекта:
1. Изучить принципы работы с HTTP/HTTPS модулями в Node.js
2. Научиться работать с переменными окружения для хранения конфиденциальных данных
3. Освоить получение данных из внешних API
4. Реализовать консольное приложение для динамической загрузки данных

### Используемый API
В качестве источника данных о погоде используется **API Яндекс.Погода** (российский аналог weatherstack.com, доступный без ограничений на территории РФ).

- Документация: [Яндекс.Погода API](https://yandex.ru/dev/weather/)
- Формат данных: JSON
- Авторизация: по ключу в заголовке `X-Yandex-Weather-Key`

## 🚀 Установка и запуск

### Требования
- Node.js (версия 12 или выше)
- npm (менеджер пакетов)

### Шаги для установки

1. **Клонировать репозиторий**
```bash
git clone https://github.com/jenycher/nodejs.git
cd nodejs
git checkout homework/http
cd weather-app
```

2. **Установить зависимости**
```bash
npm install
```

3. **Настроить переменные окружения**
   
   Создайте файл `.env` в папке `weather-app` и добавьте ваш API-ключ:
```env
YANDEX_WEATHER_API_KEY=ваш_api_ключ
```

   > **Важно:** Файл `.env` добавлен в `.gitignore` и не будет попадать в репозиторий!

4. **Запустить приложение**
```bash
node app.js <название_города>
```

### Примеры использования

```bash
# Получить погоду в Москве
node app.js moscow

# Получить погоду в Санкт-Петербурге
node app.js spb

# Получить погоду в Казани
node app.js kazan

# Получить погоду в Новосибирске
node app.js novosibirsk
```

## ⚙️ Конфигурация

### Файл `config.js`

Содержит настройки приложения и значения по умолчанию:

```javascript
const dotenv = require('dotenv');
const path = require('path');

dotenv.config({ path: path.resolve(__dirname, '.env') });

module.exports = {
    apiKey: process.env.YANDEX_WEATHER_API_KEY,
    apiUrl: process.env.API_URL || 'https://api.weather.yandex.ru/v2/forecast',
    limit: process.env.LIMIT || 1,
    hours: process.env.HOURS || false,
    cities: {
        'moscow': { lat: 55.751244, lon: 37.618423 },
        'spb': { lat: 59.934280, lon: 30.335099 },
        'kazan': { lat: 55.796127, lon: 49.106405 },
        'novosibirsk': { lat: 55.008352, lon: 82.935733 },
        'ekaterinburg': { lat: 56.838926, lon: 60.605702 },
        'sochi': { lat: 43.585525, lon: 39.723062 },
        'vladivostok': { lat: 43.115067, lon: 131.885577 }
    }
};
```

### Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `YANDEX_WEATHER_API_KEY` | API ключ для доступа к сервису | Обязательное поле |
| `API_URL` | URL API Яндекс.Погоды | `https://api.weather.yandex.ru/v2/forecast` |
| `LIMIT` | Количество дней прогноза | `1` |
| `HOURS` | Почасовой прогноз | `false` |

## 💻 Технические детали

### Используемые модули Node.js

- **`https`** - для выполнения HTTPS запросов к API Яндекс.Погоды
- **`process`** - для работы с аргументами командной строки и переменными окружения
- **`dotenv`** - для загрузки переменных из файла `.env`

### Особенности реализации

1. **Безопасность**: API ключ хранится в переменных окружения и не попадает в код
2. **Асинхронность**: Все запросы к API выполняются асинхронно с использованием событийной модели Node.js
3. **Обработка ошибок**: Приложение корректно обрабатывает:
   - Отсутствие API ключа
   - Неверное название города
   - Проблемы с сетью
   - Ошибки API (401, 404, 500 и др.)
4. **Гибкость**: Легко добавлять новые города в конфигурационный файл

## 🔧 Возможные проблемы и решения

### Проблема: "API-ключ не найден"
**Решение:** Убедитесь, что вы создали файл `.env` в папке `weather-app` и добавили в него переменную `YANDEX_WEATHER_API_KEY`

### Проблема: "Город не найден"
**Решение:** Проверьте правильность написания города. Доступные города можно посмотреть в файле `config.js`

### Проблема: "Ошибка HTTP 401"
**Решение:** Проверьте правильность API ключа. Возможно, ключ не активирован или истек

### Проблема: "Ошибка сети"
**Решение:** Проверьте интернет-соединение и доступность API Яндекс.Погоды

### Проблема: "Cannot find module 'dotenv'"
**Решение:** Установите зависимости командой `npm install` в папке `weather-app`
