# Отчёт по домашнему заданию «Потоки. Модули: path, fs»

## 📌 Выполненные задания

### ✅ Задание 1: Игра «Орёл или решка»

**Описание:**
Разработана консольная игра, в которой:
- Программа случайным образом загадывает число (1 - Орёл, 2 - Решка)
- Пользователь вводит свою догадку
- Результат партии сохраняется в JSON-файл логов
- Лог содержит временную метку и результат (win/lose)

**Файл реализации:** `game.js`

**Особенности реализации:**
- Использование модуля `readline` для интерактивного ввода
- Модуль `fs` для работы с файловой системой
- Модуль `path` для корректной работы с путями
- Валидация пользовательского ввода
- Возможность играть несколько партий подряд

**Формат лог-файла:**
```json
[
  {
    "timestamp": "2024-01-15T12:30:45.123Z",
    "result": "win"
  },
  {
    "timestamp": "2024-01-15T12:31:20.456Z",
    "result": "lose"
  }
]
```

### ✅ Задание 2: Анализатор игровых логов

**Описание:**
Разработана программа, которая:
- Принимает путь к файлу логов в качестве аргумента
- Анализирует статистику игр
- Выводит в консоль:
  - Общее количество партий
  - Количество выигранных/проигранных партий
  - Процентное соотношение выигранных партий

**Файл реализации:** `analyzer.js`

## 🚀 Инструкция по запуску

### 1. Запуск игры
```bash
node game.js logs.json
```

### 2. Запуск анализатора логов
```bash
node analyzer.js logs.json
```

## 🛠 Технологии

- **Node.js** v14+
- Встроенные модули:
  - `fs` - для работы с файловой системой
  - `path` - для работы с путями
  - `readline` - для интерактивного ввода/вывода

## 📁 Структура лог-файла

Лог-файл имеет формат JSON и содержит массив объектов с полями:
- `timestamp` - дата и время игры в формате ISO
- `result` - результат партии (`win` или `lose`)
